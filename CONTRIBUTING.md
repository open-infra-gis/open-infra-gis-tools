# Contributing

Thank you for your interest in contributing to Open Infra GIS Tools.

This project focuses on practical, reusable, and non-sensitive GIS workflows for infrastructure-scale geospatial data, environmental monitoring, public datasets, and spatial data quality.

## Contribution principles

- Use only generic, public, synthetic, or non-sensitive examples.
- Do not include corporate, private, restricted, or operational datasets.
- Do not include database credentials, internal URLs, IP addresses, tokens, usernames, passwords, or private table names.
- Prefer small tools that solve one practical GIS problem clearly.
- Add comments explaining required user settings.
- Keep examples understandable for GIS analysts as well as developers.

## Suitable contributions

- Oracle Spatial / SDO_GEOMETRY helper SQL
- QGIS Python automation scripts
- GeoServer publishing and troubleshooting notes
- WMS/WFS workflow examples
- Public geospatial data processing examples
- NASA Black Marble / VIIRS workflows
- Spatial data quality checks
- Infrastructure resilience and preventive planning workflows

## Placeholder policy

Replace sensitive details with placeholders:

```text
YOUR_SCHEMA
YOUR_TABLE
YOUR_GEOMETRY_COLUMN
YOUR_WMS_URL
YOUR_WFS_URL
YOUR_OUTPUT_FOLDER
```

## Issue format

When opening an issue, include:

- Problem or workflow description
- Expected output
- Tool or platform involved
- Example input type, if relevant
- Safety or privacy considerations
